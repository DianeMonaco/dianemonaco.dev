Introduction

Note: This guide assumes that the reader has some familiarity with programming and data types. For an introduction on these topics, A Complete Beginner's Guide to Programming by Ali Spittel is an excellent place to start.

Python is a high-level object-oriented scripting language, primarily used for server-side automation, and Python 3 is the latest major implementation. It has an emphasis on readability and follows explicit rules, particularly with whitespace characters. It was first released in 1990 and is still receiving regular updates.

Python follows 19 guiding principles outlined in "The Zen of Python," a manifesto written by Tim Peters in the public domain. It can even be accessed within the Python interpreter, as shown in the following terminal output:

Python 3.6.0 (v3.6.0:41df79263a11, Dec 23 2016, 08:06:12) [MSC v.1900 64 bit (AMD64)] on win32 Type "help", "copyright", "credits" or "license" for more information. >>> import this The Zen of Python, by Tim Peters Beautiful is better than ugly. Explicit is better than implicit. Simple is better than complex. Complex is better than complicated. Flat is better than nested. Sparse is better than dense. Readability counts. Special cases aren't special enough to break the rules. Although practicality beats purity. Errors should never pass silently. Unless explicitly silenced. In the face of ambiguity, refuse the temptation to guess. There should be one-- and preferably only one --obvious way to do it. Although that way may not be obvious at first unless you're Dutch. Now is better than never. Although never is often better than *right* now. If the implementation is hard to explain, it's a bad idea. If the implementation is easy to explain, it may be a good idea. Namespaces are one honking great idea -- let's do more of those! >>>

This is, admittedly, a lot of information to digest when you're first getting started with learning the language. The biggest thing to take away as a beginner from these principles is the second line: "Readability counts." Python (mostly) forces programmers to write readable code, to the point where the interpreter does not ignore whitespace characters. More on this in the section on Code Blocks.

Hello World (And Variables)

The best way to understand how Python emphasizes readability is to look at some example programs, and Hello World is a good place to start:

print('Hello, World!')

That's it. Really! There's no need to import any libraries, no need to declare any functions, and no unintuitively named methods to call. You don't even need to add a semicolon at the end of each line! Above almost everything else, Python prioritizes code being readable.

Declaring variables is similarly straightforward. The following program will output the first sentence of Mary Shelly's Frankensein, followed by the number 8:

frankenstein = "I am by birth a Genevese, and my family is one of the most distinguished of that republic." print(frankenstein) myVar1 = 5 myVar2 = 3 print(myVar1 + myVar2)

An important thing to note about Python is that unlike JavaScript, it does not coerce variables into the same type when presented with two different types of variables. For example, if we were to type print(3 + "5") into our Python interpreter, we would get an error.

F-Strings

One of the distinguishing features of Python 3 is how it handles formatting variables into strings. Python 3 uses a data type called "f-strings" when inserting variables into strings. F-strings have special syntax compared with normal strings. As an example, the following outputs this page's author's name and age:

>>> name = "Diane" >>> age = 30 >>> print(f"{name} is {age} years old and says hello!") Diane is 30 years old and says hello!

The syntax for f-strings is a lower or uppercase 'f' followed by single or double quotes, with variable names surrounded by {curly brackets}. F-strings can be used anywhere regular strings can be used, and can even be used inside variable declarations.

Python 3 still supports the older method of formatting strings by using str.format() but this method is much less readable to anyone viewing program code. The following example has the same output as the previous one, but uses the older method of str.format():

>>> name = "Diane" >>> age = 30 >>> print("{} is {} years old and says hello!".format(name, age)) Diane is 30 years old and says hello!

Formatting strings by using str.format() is less common in Python 3 programs, but it is still important to recognize it, as it often appears in legacy code.

Code Blocks

One of the unique (and controversial) things about Python is how it handles code blocks and whitespace characters. The syntax for code blocks is influenced by a few of the guiding principles outlined earlier. In particular:

  • Beautiful is better than ugly.
  • Explicit is better than implict.
  • Sparse is better than dense.
  • Readability counts.

The syntax for code blocks follows three rules:

  1. The line of code that starts a block ends with a colon. (e.g. if age == 30:)
  2. Each line of code in a block is preceded by exactly four (4) whitespace characters.
  3. The last line of code with 4 preceding whitespace characters is the last line of the block.

Blocks of code can be nested within each other, but it's considered best practice to avoid this when possible. The following is an example of Python's syntax for code blocks, including a nested block of code.

print("Let's see what numbers up to 20 are divisible by 5.") i = 0 while i <= 20: if i % 5 == 0: print(f"{i} is divisible by 5.") else: print(f"{i} is not divisible by 5.") print("Let's move on to the next number.") i += 1 print("Those are all the positive numbers up to 20 that are divisble by 5.")

Although whitespace characters are explicit in Python, extra return carriages are not. Code within a block can have extra return carriages and will still execute properly, as long as the correct number of whitespace characters are used.

Functions

The syntax for declaring and calling functions is straightforward and consistent in Python. A function will always be declared by a def statement, followed by the function's name, followed by any arguments it will take (contained within parentheses), and ending with a colon (:). Functions do not need to have a return statement in order to end and will end whenever their code block does. Functions must be declared before they are called. An example program with a function that greets the user follows: def sayHello(name): print(f"Hello, {name}.") print("It's a pleasure to meet you!") print("Please enter your name.") sayHello(input("> "))

When the author of this page runs the program, her terminal output looks like this:

Please enter your name. > Diane Hello, Diane. It's a pleasure to meet you!

There is no set upper or lower limit to the number of arguments a function can accept in Python, but for the sake of readability, it's generally considered best practice to limit the maximum number of arguments to three. Functions can be called within other functions as long as they're defined before they're called.
